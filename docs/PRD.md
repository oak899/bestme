# GrowthOS / 成长计划 — 产品需求文档 (PRD)

**版本:** 1.0  
**产品名:** GrowthOS（成长计划 / Personal Growth Planner）  
**定位:** 个人版「Jira + 日历 + 复盘」— 每日计划、任务流转、工时统计、成长报表

---

## 1. 产品愿景

帮助知识工作者与自驱型用户：**每天有计划、执行可追踪、时间可量化、成长可复盘**。比 Jira 更轻、比 Todo 更专业，结合 AI 教练降低规划成本。

## 2. 目标用户

- 自由职业者、创业者、独立开发者
- 需要自我管理的学生与职场人
- 希望用数据驱动个人成长的用户

## 3. 核心价值

| 价值 | 说明 |
|------|------|
| 计划 | 每日重点 + 任务清单 + 预计/实际工时 |
| 执行 | Jira 式状态流转 + Kanban + 计时器 |
| 洞察 | 日报/周报/月报 + 图表 + AI 复盘 |
| 提醒 | 时间/地点/番茄钟/超工时提醒 |

## 4. 功能需求

### 4.1 首页 Dashboard

| 模块 | 需求 | 优先级 |
|------|------|--------|
| 今日计划摘要 | 展示当日 Daily Plan 重点目标 | P0 |
| 待办任务 | status ∈ {Backlog, Todo}，按优先级排序 | P0 |
| 进行中 | status = In Progress | P0 |
| 已完成 | status = Done，今日完成 | P0 |
| 今日工作总时长 | 当日 time_entries 合计 | P0 |
| 本周目标完成率 | 本周完成任务数 / 本周计划任务数 | P1 |
| 每日激励语 | 本地库 + AI 可选 | P1 |

### 4.2 每日计划 Daily Plan

- 每天一条 `daily_plans` 记录（用户 + 日期唯一）
- 字段：重点目标、任务列表（关联）、预计工时、实际工时、复盘、明日改进
- 复制昨日计划
- AI 生成今日计划建议（基于 backlog + 历史）

### 4.3 任务系统（类 Jira）

**Task 字段:** 标题、描述、优先级(High/Medium/Low)、状态、截止日期、预计/实际耗时、标签、所属项目、父任务 ID

**状态流转:**

```
Backlog → Todo → In Progress → Done
              ↘ Blocked ↗
```

**能力:** Kanban 拖拽、评论、子任务、状态变更历史（audit log）

### 4.4 项目管理

- 多项目：名称、目标、起止日期、进度%、关联任务
- 项目页：任务列表、时间投入、完成率

### 4.5 日历

- 日/周/月视图
- 任务时间块、Daily Plan 锚点
- 拖拽调整（MVP 后可做）

### 4.6 提醒

| 类型 | MVP | 完整版 |
|------|-----|--------|
| 任务截止提醒 | P1 | ✓ |
| 每日计划提醒 | P1 | ✓ |
| 番茄钟 25/50min | P2 | ✓ |
| 超工时复盘提醒 | P2 | ✓ |
| 地理位置打卡 | P3 | ✓ |

### 4.7 工作时长

- 任务计时器：开始/暂停/继续/结束 → `time_entries`
- 统计：日/周/月、按项目/标签分布

### 4.8 报表

- 每日完成率、周趋势、月工时、项目占比、高优完成率、拖延任务数、工作时段热力图
- 图表：折线、柱状、饼图、热力（MVP 先做 3 种）

### 4.9 AI Coach

| 能力 | 输入 | 输出 |
|------|------|------|
| 生成今日计划 | 自然语言 + 任务列表 | Daily Plan + Tasks |
| 每日复盘 | 当日任务与工时 | 复盘文案 |
| 拖延分析 | 未完成/逾期任务 | 原因与建议 |
| Top 3 明日任务 | 历史 + backlog | 3 条任务建议 |
| 周报 | 一周数据 | 成长总结 |
| NL → Task | 自然语言 | 结构化 Task |

### 4.10 设置

- 每日目标工时、提醒时间、常用地点、工作日、默认优先级、深色模式、成长目标

## 5. 非功能需求

- **性能:** API P95 < 300ms（除 AI）
- **安全:** JWT 登录、HTTPS、密码 bcrypt
- **平台:** iOS / Android / Web（Flutter）
- **离线:** 本地缓存（Hive/Drift），联网同步
- **扩展:** Google/Apple Calendar、OpenAI（已接）

## 6. 技术栈

| 层 | 选型 | 说明 |
|----|------|------|
| 前端 | Flutter 3.x | Material 3，移动端优先 |
| 后端 | Go 1.20+ | REST API，chi router |
| 数据库 | **PostgreSQL**（生产）/ SQLite（MVP 单机） |  schema 兼容，便于迁移 |
| 缓存 | 客户端 Hive | 设置、草稿、离线队列 |
| AI | OpenAI API | gpt-4o-mini |
| 部署 | 45.76.66.28 | bestme.zfloo.com（规划） |

## 7. 竞品差异

- **Jira:** 太重，无个人成长叙事
- **Notion:** 自由度高但无原生工时/Kanban 流转
- **GrowthOS:** 个人单用户、计划+执行+报表闭环、AI 教练内置

## 8. 成功指标 (MVP 后 4 周)

- 日活用户完成 ≥1 个任务/天
- 日均计时 ≥ 30 分钟
- AI 计划使用率 ≥ 20%
- 崩溃率 < 0.5%

## 9. 范围外 (V2+)

- 多人协作、团队空间
- 第三方 Calendar 双向同步
- 原生地理围栏（需平台插件）
